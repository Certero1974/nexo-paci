# Documento Maestro de Rediseño UX
## Design Thinking · Human Centered Design

> PACIA no es un sistema para completar formularios. Es un espacio de trabajo inteligente que acompaña a los equipos PIE para dedicar menos tiempo a la burocracia y más tiempo a las personas.

# Visión

Cada decisión pedagógica debe ser más simple, colaborativa y basada en evidencia.

La tecnología existe para disminuir la carga administrativa, nunca para aumentarla.

## Principio rector

**Menos papeleo. Más inclusión.**

Toda funcionalidad debe responder:

> ¿Esta funcionalidad libera tiempo para trabajar con estudiantes?

Si la respuesta es "no", debe rediseñarse.

# Design Thinking

## Humanamente deseable

- Reducir estrés.
- Reducir carga cognitiva.
- Facilitar la colaboración.
- Mostrar solo información útil.
- Hacer sentir acompañado al usuario.
- Priorizar la acción sobre la burocracia.

Cada pantalla debe responder:

**¿Qué necesita esta persona en este momento?**

## Técnicamente posible

Implementable con:

- React / Next.js
- FastAPI
- PostgreSQL
- OCR
- IA Generativa
- RAG
- Permisos por rol

## Económicamente viable

- Arquitectura modular.
- IA bajo demanda.
- Escalable.
- Reutilización de análisis.

# Nuevo paradigma

Persona → Estudiante → Evidencias → Equipo → Decisiones → Documento

El documento es el resultado, nunca el centro.

# Dashboard

Eliminar estadísticas decorativas.

Mostrar:

- Buenos días.
- Tareas prioritarias.
- Tiempo estimado.
- Documentos nuevos.
- Estudiantes críticos.
- Próximo paso.

Responder siempre:

¿Qué hago ahora?

# Estudiante al centro

Mostrar primero:

- Foto
- Nombre
- Curso
- Fortalezas
- Necesidades
- Equipo
- Próxima acción

Después los documentos.

# Expediente vivo

Usar línea de tiempo:

Ingreso PIE → Evaluaciones → Informes → Observaciones → Adecuaciones → Validaciones → Seguimiento → PACI

# Copiloto PACIA

La IA debe:

- resumir
- explicar
- justificar
- detectar contradicciones
- encontrar evidencias
- sugerir próximos pasos

Siempre mostrando origen, fecha, profesional y evidencia.

# Centro de colaboración

Cada estudiante tendrá un espacio donde el equipo pueda:

- comentar
- validar
- responder
- adjuntar evidencia
- acordar decisiones

La IA resumirá acuerdos y pendientes.

# Navegación

Cada pantalla debe responder:

- ¿Qué debo hacer?
- ¿Por qué?
- ¿Cuánto demora?
- ¿Quién espera mi respuesta?
- ¿Qué sigue?

# Información relevante

Mostrar solo información accionable:

- tareas
- pendientes
- documentos nuevos
- estudiantes sin seguimiento
- observaciones sin responder

# Diseño emocional

Transmitir:

- calma
- claridad
- confianza
- cercanía
- acompañamiento

Nunca sensación de auditoría.

# Métricas

No medir formularios.

Medir:

- horas ahorradas
- colaboración
- satisfacción
- tiempo recuperado
- participación interdisciplinaria

# Manifiesto

PACIA nace para devolver tiempo a quienes acompañan a estudiantes.

La burocracia nunca debe ocupar el lugar de la pedagogía.

Cada minuto que ahorra PACIA vuelve al estudiante.
